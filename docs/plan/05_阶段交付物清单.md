# 网上书店系统 - 阶段交付物清单

## 1. 软件工程标准流程

```
需求分析 → 概要设计 → 详细设计 → 编码实现 → 测试 → 项目答辩
    ↓           ↓           ↓           ↓        ↓       ↓
  文档1       文档2       文档3       源码     测试报告  演示
```

## 3. 常用图表说明

### 3.1 用例图（USE CASE）

**工具**：StarUML、Visio、ProcessOn、draw.io

**内容**：
- 参与者（Actor）
- 用例（Use Case）
- 参与者与用例的关系
- 系统边界

**适用场景**：需求分析阶段，展示系统功能全景

```
┌─────────────────────────────────┐
│        网上书店系统             │
│                                 │
│  ┌─────┐    ┌──────────────┐  │
│  │用户 │───→│  浏览书籍     │  │
│  │     │    │  搜索书籍     │  │
│  │     │    │  加入购物车   │  │
│  │     │    │  提交订单     │  │
│  └─────┘    └──────────────┘  │
│       │                          │
│  ┌─────┴──────┐                 │
│  │  管理员   │──→  后台管理    │
│  └───────────┘                 │
└─────────────────────────────────┘
```

### 3.2 ER图

**工具**：PowerDesigner、Navicat、StarUML、ProcessOn

**内容**：
- 实体（Entity）
- 属性（Attribute）
- 主键（Primary Key）
- 关系（Relationship）

**示例**：
```
┌─────────┐       ┌─────────┐
│  User   │       │  Book   │
├─────────┤       ├─────────┤
│ id (PK) │←──1:N─│ id (PK) │
│username │       │ title   │
│password │       │ price   │
│ email   │       │ ...     │
└─────────┘       └─────────┘
```

### 3.3 类图

**工具**：StarUML、Visio、IntelliJ IDEA插件

**内容**：
- 类名
- 属性（可见性、类型）
- 方法（可见性、参数、返回值）
- 类之间的关系

**示例**：
```
┌─────────────────────────┐
│        User             │
├─────────────────────────┤
│ - id: Long              │
│ - username: String      │
│ - password: String      │
│ - email: String         │
├─────────────────────────┤
│ + login(): boolean      │
│ + register(): void      │
│ + updateInfo(): void    │
└─────────────────────────┘
```

### 3.4 时序图（顺序图）

**工具**：StarUML、Visio、ProcessOn

**内容**：
- 对象（Object）
- 生命线（Lifeline）
- 消息（Message）
- 激活条（Activation）

**示例**：
```
用户          系统           数据库
  │             │              │
  │──登录请求──→│              │
  │             │──查询用户───→│
  │             │←─用户信息────│
  │←─登录结果──│              │
```

### 3.5 状态图

**工具**：StarUML、Visio、ProcessOn

**适用场景**：订单状态、用户状态等有明确状态流转的场景

**示例（订单状态）**：
```
┌────────┐  支付   ┌────────┐  发货   ┌────────┐  收货   ┌────────┐
│待支付  │──────→│已支付  │──────→│已发货  │──────→│已收货  │──────→完成
└────────┘        └────────┘        └────────┘        └────────┘
     │                │                 │
     ↓ 取消           ↓ 退款           ↓ 退货
┌────────┐        ┌────────┐        ┌────────┐
│已取消  │        │已退款  │        │已退货  │
└────────┘        └────────┘        └────────┘
```

## 5. 文档撰写规范

### 5.1 文档格式

- **字体**：宋体（正文）/ 黑体（标题）
- **字号**：正文小四，标题三号/四号
- **行距**：1.5倍行距
- **页眉**：项目名称 + 章节名称
- **页脚**：页码（居中）

### 5.2 文档结构

```
封面
    项目名称
    课程设计报告
    班级、学号、姓名、日期

目录

一、 引言
    1.1 项目背景
    1.2 目的和意义

二、 需求分析
    ...

三、 概要设计
    ...

四、 详细设计
    ...

五、 编码实现
    ...

六、 测试
    ...

七、 总结
    ...

参考文献

附录
```

## 6. 答辩PPT结构建议（10-15页）

1. **封面** - 项目名称、团队成员
2. **目录** - 汇报大纲
3. **项目背景** - 为什么做这个项目
4. **需求分析** - 核心功能、用例图
5. **系统设计** - 架构图、ER图
6. **数据库设计** - 表结构
7. **详细设计** - 核心类图、时序图
8. **系统实现** - 关键代码、界面截图
9. **测试结果** - 测试用例、结果
10. **总结与展望** - 收获、不足、改进
11. **致谢**
