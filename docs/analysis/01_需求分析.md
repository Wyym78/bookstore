# 网上书店系统需求分析报告

---

## 1. 项目概述

### 1.1 项目背景

网上书店系统是一个面向读者的 B2C 电子商务平台，提供图书浏览、购买、评论等核心功能。系统采用前后端分离架构，后端基于 Spring Boot 3 + MyBatis-Plus，前端基于 Vue 3 + Vite。

### 1.2 项目目标

- 为用户提供便捷的图书购买体验
- 为管理员提供完善的后台管理功能
- 实现安全的用户认证与权限控制
- 保证数据一致性与系统稳定性

---

## 2. 用户角色分析

| 角色 | 说明 | 权限范围 |
|------|------|---------|
| 游客 | 未登录用户 | 浏览书籍、搜索、注册 |
| 普通用户 | 已登录的非管理员 | 购物车、订单、评论、个人中心、收货地址管理 |
| 管理员 | 系统维护人员 | 后台管理全功能（图书/分类/订单/用户/评论管理） |

---

## 3. 功能需求分析

### 3.1 用户模块

| 功能 | 需求描述 | 接口路径 | 请求方式 |
|------|---------|---------|---------|
| 用户注册 | 输入用户名、密码、邮箱/手机号创建账号 | `/api/user/register` | POST |
| 用户登录 | 输入账号密码登录，返回 JWT Token | `/api/user/login` | POST |
| 个人信息查看 | 获取当前用户基本信息 | `/api/user/info` | GET |
| 个人信息修改 | 修改邮箱、手机号等个人信息 | `/api/user/update` | PUT |
| 密码修改 | 修改当前用户密码 | `/api/user/password` | PUT |

**业务规则：**
- 用户名唯一，长度 3-20 位
- 密码使用 BCrypt 加密存储
- JWT Token 有效期 24 小时
- 登录失败返回统一错误信息，不暴露具体原因

### 3.2 书籍模块

| 功能 | 需求描述 | 接口路径 | 请求方式 |
|------|---------|---------|---------|
| 书籍分页列表 | 获取上架书籍分页列表 | `/api/book/list` | GET |
| 书籍详情 | 获取书籍详细信息含评分评论 | `/api/book/{id}` | GET |
| 关键字搜索 | 按标题/作者/ISBN 搜索书籍 | `/api/book/search` | GET |
| 分类浏览 | 按分类 ID 筛选书籍 | `/api/book/category/{categoryId}` | GET |

**业务规则：**
- 只显示 status=1（上架）的书籍
- 搜索支持标题、作者、ISBN 模糊匹配
- 详情页返回平均评分和评论数

### 3.3 分类模块

| 功能 | 需求描述 | 接口路径 | 请求方式 |
|------|---------|---------|---------|
| 分类树形结构 | 获取所有分类（支持层级） | `/api/category/tree` | GET |
| 分类详情 | 获取指定分类信息 | `/api/category/{id}` | GET |

**业务规则：**
- 分类支持树形结构（parent_id 区分层级）
- 删除分类前检查是否有子分类或关联书籍

### 3.4 购物车模块

| 功能 | 需求描述 | 接口路径 | 请求方式 |
|------|---------|---------|---------|
| 加入购物车 | 添加书籍到购物车 | `/api/cart/add` | POST |
| 购物车列表 | 获取当前用户购物车 | `/api/cart/list` | GET |
| 修改数量 | 更新购物车商品数量 | `/api/cart/update` | PUT |
| 删除商品 | 从购物车移除商品 | `/api/cart/{id}` | DELETE |

**业务规则：**
- 同一用户对同一书籍只能有一条购物车记录
- 数量必须大于 0
- 下架书籍无法加入购物车

### 3.5 订单模块

| 功能 | 需求描述 | 接口路径 | 请求方式 |
|------|---------|---------|---------|
| 创建订单 | 从购物车选择商品下单 | `/api/order/create` | POST |
| 订单支付 | 模拟支付订单 | `/api/order/{id}/pay` | PUT |
| 订单取消 | 取消未支付订单 | `/api/order/{id}/cancel` | PUT |
| 确认收货 | 用户确认收到货物 | `/api/order/{id}/confirm` | PUT |
| 订单列表 | 获取当前用户订单分页列表 | `/api/order/list` | GET |
| 订单详情 | 获取订单详细信息 | `/api/order/{id}` | GET |

**订单状态流转：**
```
pending（待支付）→ paid（已支付）→ shipped（已发货）→ delivered（已发货）→ completed（已完成）
     ↓                ↓
 cancelled（已取消）  cancelled（已取消）
```

**业务规则：**
- 创建订单时原子性扣减库存（防止超卖）
- 取消订单时回滚库存
- 只能查看和操作本人的订单

### 3.6 评论模块

| 功能 | 需求描述 | 接口路径 | 请求方式 |
|------|---------|---------|---------|
| 发表评论 | 对书籍进行评分和评论 | `/api/review/add` | POST |
| 书籍评论列表 | 获取书籍的所有显示中评论 | `/api/review/book/{bookId}` | GET |
| 删除评论 | 删除本人评论或管理员删除 | `/api/review/{id}` | DELETE |

**业务规则：**
- 评论状态为显示（status=1）才可见
- 评论删除需本人或管理员权限

### 3.7 收货地址模块

| 功能 | 需求描述 | 接口路径 | 请求方式 |

## 5. 安全需求

### 5.1 认证与授权

- 所有 `/api/*` 接口（除登录注册）需要携带有效 JWT Token
- `/admin/*` 接口需要管理员角色权限
- JWT 密钥存储在配置文件中，不硬编码

### 5.2 密码安全

- 用户密码使用 BCrypt 单向加密存储
- BCrypt 内置盐值，防范彩虹表攻击

### 5.3 数据完整性

- 库存扣减使用原子性 SQL 操作
- 订单创建使用事务保证数据一致性
- 删除关联数据前进行业务校验

## 8. 项目亮点

1. **原子性库存扣减**：避免超卖问题
2. **层级分类设计**：支持无限层级分类树
3. **JWT Token 认证**：无状态认证，易于扩展
4. **BCrypt 密码加密**：企业级密码安全
5. **RESTful API 设计**：统一响应格式，易于调用
